# 4. Why Mounting is Needed

Docker containers are isolated.

Files from local machine are NOT automatically visible inside container.

So we mounted folders using docker-compose.yml.

Example:

volumes:
  - ./ml:/opt/airflow/ml
  - ./data:/opt/airflow/data

Meaning:

Local Folder                -> Container Folder
./ml                        -> /opt/airflow/ml
./data                      -> /opt/airflow/data

---

# 5. How to Check if Mounting Worked

Run:

docker exec -it mlops-project-airflow-scheduler-1 ls /opt/airflow

Expected output:

config  dags  data  logs  ml  plugins

Then check inside folder:

docker exec -it mlops-project-airflow-scheduler-1 ls /opt/airflow/data

Expected:

housing.csv

---

# 6. Why PYTHONPATH was Needed

Even after mounting, Python could not import:

from ml.src.data_ingestion import load_data

Because Python did not search /opt/airflow automatically.

So we added:

PYTHONPATH=/opt/airflow

inside .env file.

This tells Python:

"Search /opt/airflow for modules/packages"

---

# 7. Why __init__.py was Needed

Python treats folders as packages only if __init__.py exists.

Created:

ml/__init__.py
ml/src/__init__.py

Without these files:
import statements fail.

---

# 9. Important Learning
